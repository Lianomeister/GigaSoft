# GigaSoft Software Setup AI Blueprint

## Zweck
Umsetzungsplan fuer die standalone-first Weiterentwicklung von GigaSoft.

## Aktive Zielarchitektur
- `gigasoft-api`
- `gigasoft-host-api`
- `gigasoft-net`
- `gigasoft-runtime`
- `gigasoft-core`
- `gigasoft-standalone`
- `gigasoft-demo-standalone`

## Leitprinzipien
1. Standalone zuerst.
2. Host-neutrale Kernlogik.
3. Sicherheit standardmaessig.
4. Performance messbar halten.

## Build/Test Gates
1. `gradlew test`
2. `gradlew performanceBaseline`
3. `gradlew standaloneReleaseCandidate`

## Definition of Done fuer Releases
- alle Tests gruen
- `performanceBaseline` gruen
- `standaloneReleaseCandidate` gruen
- Release aus `build/release-standalone/<version>/`
