# Clockwork Software Setup AI Blueprint

## Zweck
Umsetzungsplan fuer die standalone-first Weiterentwicklung von Clockwork.

## Aktive Zielarchitektur
- `clockwork-api`
- `clockwork-host-api`
- `clockwork-net`
- `clockwork-runtime`
- `clockwork-core`
- `clockwork-standalone`
- `clockwork-demo-standalone`

## Leitprinzipien
1. Standalone zuerst.
2. Host-neutrale Kernlogik.
3. Sicherheit standardmaessig.
4. Performance messbar halten.

## Build/Test Gates
1. `gradlew test`
2. `gradlew performanceBaseline`
3. `gradlew standaloneReleaseCandidate`

## Definition of Done fuer Releases
- alle Tests gruen
- `performanceBaseline` gruen
- `standaloneReleaseCandidate` gruen
- Release aus `build/release-standalone/<version>/`
